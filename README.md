# ZS-Services

🧾 0. Общие требования (обязательные для всех)
==============================================

**Тип проекта:**

*   Одностраничный лендинг (SPA/SSG через Next.js static export)
    

**Цель:**

*   Максимальная конверсия в:
    
    *   звонок
        
    *   сообщение (WhatsApp/Telegram)
        
    *   отправку формы
        

**ЦА:**

*   частные клиенты (квартиры/дома)
    
*   низкий уровень доверия → нужен упор на “живость” и прозрачность
    

**Основной принцип:**

> Минимум текста → максимум доверия и конкретики

* * *

🎨 1. Дизайн-система (для дизайнера + фронтенда)
================================================

1.1 Цвета (строго)
------------------

Основываясь на логотипе:

**Primary palette:**

*   Синий (основной): #1E3A8A (или близкий)
    
*   Голубой: #0EA5E9
    
*   Зелёный: #22C55E
    
*   Красный/акцент: #EF4444
    
*   Жёлто-оранжевый: #F59E0B
    

**Background:**

*   Основной: #FFFFFF
    
*   Вторичный: #F8FAFC
    

**Правило:**

*   НЕ использовать все цвета одновременно в одном блоке
    
*   1 основной + 1 акцент
    

* * *

1.2 Типографика
---------------

**Стиль:**

*   базовый текст — нейтральный (чёрный/тёмно-серый)
    
*   заголовки — с цветными акцентами (как “мазки краски”)
    

**Шрифты:**

*   Заголовки: Inter / Poppins
    
*   Текст: Inter
    

**Особенность:**

*   выделения цветом (под слова, не весь текст)
    
*   имитация “покрашенности”:
    
    *   градиенты
        
    *   псевдо underline brush
        

* * *

1.3 UI стиль
------------

*   скругления: `rounded-xl`
    
*   тени: мягкие (shadow-md / shadow-lg)
    
*   кнопки:
    
    *   primary (цвет логотипа)
        
    *   secondary (outline)
        

* * *

⚙️ 2. Технический стек (строго)
===============================

*   Next.js (SSG)
    
*   Tailwind CSS
    
*   Framer Motion
    
*   Swiper
    
*   Formspree
    

* * *

🧱 3. Структура страницы (строгая)
==================================

* * *

3.1 HERO
--------

**Состав:**

*   H1 (оффер + гео)
    
*   подзаголовок
    
*   CTA (2 кнопки)
    
*   микро-доверие (3 пункта)
    

**Контент:**

*   “Ремонт квартир в \[городе\] за 7–14 дней”
    
*   “Частный мастер с опытом X лет”
    

**CTA:**

*   Позвонить (tel:)
    
*   Написать (WhatsApp)
    

**Микро-доверие:**

*   “Выезд сегодня”
    
*   “Бесплатная оценка”
    
*   “Гарантия X лет”
    

**UX:**

*   CTA всегда виден без скролла
    

* * *

3.2 БЛОК ДОВЕРИЯ
----------------

**Состав:**

*   фото мастера (реальное)
    
*   текст “кто я”
    
*   цифры (3–4)
    

**Цифры:**

*   лет опыта
    
*   объектов
    
*   % довольных клиентов
    
*   гарантия
    

**Формат:**

*   карточки или inline
    

* * *

3.3 УСЛУГИ
----------

**Кол-во:**

*   5–8 карточек
    

**Формат карточки:**

*   заголовок (результат)
    
*   короткое описание
    
*   иконка или мини-иллюстрация
    

**Пример:**

*   “Ровные стены под покраску за 2 дня”
    
*   “Укладка плитки без перепадов”
    

* * *

3.4 ГАЛЕРЕЯ (ключевой блок)
---------------------------

**Тип:**

*   Swiper carousel
    

**Контент:**

*   до / после
    
*   подпись
    
*   срок выполнения
    

**Формат:**

*   2 изображения рядом (before/after)
    
*   или слайдер с переключением
    

**Обязательно:**

*   lazy load
    
*   оптимизация изображений
    

* * *

3.5 ОТЗЫВЫ
----------

**Формат:**

*   карточки
    

**Содержимое:**

*   имя
    
*   город
    
*   текст
    

**Дополнительно:**

*   фото объекта ИЛИ
    
*   скриншот (WhatsApp/OLX)
    

* * *

3.6 CTA (повторный)
-------------------

После галереи:

*   текст: “Хотите так же?”
    
*   кнопки:
    
    *   позвонить
        
    *   написать
        

* * *

3.7 КОНТАКТЫ
------------

**Состав:**

*   телефон (кликабельный)
    
*   WhatsApp
    
*   Telegram
    
*   зона работы
    

* * *

3.8 ФОРМА
---------

**Поля:**

*   имя
    
*   телефон
    
*   комментарий (опционально)
    

**Интеграция:**

*   Formspree
    

**Поведение:**

*   success state
    
*   ошибка
    

* * *

💻 4. Фронтенд требования
=========================

4.1 Производительность
----------------------

*   Lighthouse ≥ 90
    
*   LCP < 2.5s
    

* * *

4.2 Оптимизация
---------------

*   next/image
    
*   lazy loading
    
*   минимизация JS
    

* * *

4.3 Анимации (Framer Motion)
----------------------------

Разрешено:

*   fade-in при скролле

*   micro interactions
    

Запрещено:

*   тяжёлые анимации
    
*   блокирующие рендер
    

* * *

4.4 Адаптив
-----------

*   mobile-first
    
*   брейкпоинты:
    
    *   sm
        
    *   md
        
    *   lg
